# techpawz R2 404 원인 확정 — `img.techpawz.com` 커스텀 도메인 미바인딩

**진단 일시:** 2026-07-23
**진단 범위:** 코드 수정 없음, 읽기/조회만
**원인 분류:** (a) 인프라 — Cloudflare R2 커스텀 도메인 설정

---

## 1. R2 업로드 코드 분기 확인

**확인:** 코드 분기 없음. 3개 사이트 동일 로직.

`image/r2_uploader.py` `HUGO_R2_DOMAINS` 매핑:

| 사이트 | R2 prefix | 커스텀 도메인 |
|--------|-----------|-------------|
| rotcha | `images/rotcha` | `https://img.rotcha.kr` |
| informationhot | `images/informationhot` | `https://img.informationhot.kr` |
| techpawz-hugo | `images/techpawz` | `https://img.techpawz.com` |

**버킷:** 3개 사이트 전부 동일 버킷 `hotissue-images` 사용 (환경변수 `R2_BUCKET_NAME`). 사이트별 분기 없음.

---

## 2. R2 URL 생성 로직

**확인:** 동일 로직. `upload_all_images()`가 `{r2_domain}/{r2_key}` 형식으로 URL 생성.

```
URL = f"{r2_domain}/{r2_key}"
     = "https://img.techpawz.com" + "/" + "images/techpawz/{slug}/{filename}"
```

rotcha/infohot/techpawz 전부 동일 패턴. 사이트별 차이 없음.

---

## 3. R2 버킷 퍼블릭 접근 비교

**핵심 검증:**

| URL 패턴 | rotcha | infohot | techpawz |
|----------|--------|---------|----------|
| `pub-*.r2.dev/{key}` (R2.dev 공개) | **200** ✅ | **200** ✅ | **200** ✅ |
| `img.{site}.kr/{key}` (커스텀 도메인) | **200** ✅ | **200** ✅ | **404** ❌ |

**확인:** R2.dev 공개 URL은 3개 파일 전부 HTTP 200. techpawz 파일도 R2 버킷에 존재하며 퍼블릭 액세스 가능.

---

## 4. 원인 확정 — 커스텀 도메인 미바인딩

**원인:** `img.techpawz.com` 커스텀 도메인이 R2 버킷 `hotissue-images`에 바인딩되지 않음.

**증거:**
- R2 버킷 `hotissue-images`에 techpawz 파일 존재: `images/techpawz/ai프롬프트마켓-20260722-s3/ai프롬프트마켓-20260722-s3_1024x1024.webp` (66,314 bytes)
- R2.dev 공개 URL `pub-2f5c7af1c303419a933069212bc25874.r2.dev/images/techpawz/...` → HTTP 200
- 커스텀 도메인 `img.techpawz.com/images/techpawz/...` → HTTP 404
- rotcha/infohot은 동일 구조로 동작 → 코드 문제 아님

**rotcha/infohot은 왜 되는가:**
- `img.rotcha.kr` → Cloudflare R2 커스텀 도메인에 `hotissue-images` 버킷 바인딩됨 ✅
- `img.informationhot.kr` → 동일 버킷에 바인딩됨 ✅
- `img.techpawz.com` → **미바인딩** ❌

---

## 5. 대표 수정 방법 (Cloudflare 대시보드)

Cloudflare 대시보드 → R2 → `hotissue-images` 버킷 → Settings → Public Access:

1. **Custom Domains** 탭에서 `img.techpawz.com` 추가
2. DNS 레코드 확인: `img.techpawz.com` → CNAME → `pub-2f5c7af1c303419a933069212bc25874.r2.dev`
3. SSL/TLS: Full (Strict) 권장
4. 바인딩 완료 후 `img.techpawz.com/images/techpawz/...` → HTTP 200 예상

---

## 6. 잔존 위험

- **techpawz 본문 이미지**: 커스텀 도메인 바인딩 전까지 `img.techpawz.com` 경로 404 지속
- **R2.dev 경로로 임시 대체 가능**: `pub-2f5c7af1c303419a933069212bc25874.r2.dev/images/techpawz/...` → 200 (하지만 URL이 길고 비공식)
- **rotcha/infohot**: 정상 동작, 추가 조치 불필요
